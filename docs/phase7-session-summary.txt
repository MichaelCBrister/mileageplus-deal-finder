Date: 2026-03-25

OBJECTIVE
---------
Wire the purchase log so that after a user acts on a scoring recommendation,
they can record the purchase and track whether miles post. Specifically:
  - Add POST/GET/PATCH/DELETE endpoints for purchase_log in the Node bridge
  - Add log_template field to /score and /rank responses in the bridge layer
  - Add Purchases tab to React frontend with history table, summary stats,
    manual log form, and mark-posted/delete actions
  - Add "Log this purchase" button to Score tab and "Log" buttons to Rank rows
  - No Julia engine changes — all purchase log logic lives in the Node bridge

IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /usr/local/bin/julia (Julia 1.12.5)
   Always run with: JULIA_PKG_SERVER=""

2. Database initialization: rm -f db/mileageplus.db && bash db/init.sh
   Must delete existing DB first for clean seed data.

3. The bridge converts tax_rate from percentage to decimal for /api/score
   and /api/rank. The log_template computation in the bridge also uses
   this converted decimal tax_rate for p_card calculation.

4. TIMESTAMP FORMAT: purchase-log.js formats timestamps as
   "YYYY-MM-DDTHH:MM:SS" (no milliseconds, no Z suffix) to match the
   Julia engine's dateformat parser. This is consistent with the scraper
   timestamp format from Phase 6.

5. POSTING STATUS: posting_status is derived at query time in listPurchases:
   - "posted" if miles_posted is not null
   - "overdue" if purchased_at > 90 days ago and miles_posted is null
   - "pending" otherwise
   The 90-day threshold comes from v3-spec.md section 3.4. Some retailers
   may take up to 120 days. The threshold is documented in a code comment.

6. SPEND VECTOR DEFAULTS: When POST /api/purchases receives only p_list
   (without p_portal, p_card, p_cash), the bridge fills them all with
   p_list value. This is the simple default for the manual log form.
   The log_template from /score and /rank uses proper tax-adjusted values.

7. The log_template is computed in the bridge layer, NOT in the Julia engine.
   Julia does not know about the purchase log. This keeps the separation
   clean: Julia scores, bridge manages purchase tracking.

8. The "Log" button in the Rank table briefly highlights the row green for
   2 seconds after a successful log. This is a CSS transition, not a
   timer-based poll.
