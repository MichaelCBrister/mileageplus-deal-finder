FILES CREATED / MODIFIED
--------------------------

bridge/server.js  [NEW]
  Express server on port 4000. Endpoints:
    GET  /health     -> bridge + Julia status
    POST /api/score  -> forwards to Julia engine with 2s timeout
    POST /api/rank   -> 501 stub (Phase 4)
  Reads JULIA_ENGINE_URL and BRIDGE_PORT from env vars.
  Maps frontend card_tier values (none/one_x/one_five_x/two_x) to
  Julia engine keys (none/explorer/club/quest).
  Converts frontend tax_rate percentage to decimal for Julia.
  Request logging middleware (method, path, status, duration).

bridge/package.json  [MODIFIED]
  Updated main from src/index.js to server.js.
  Updated scripts to use server.js directly.
  Version bumped to 0.2.0.

bridge/.env.example  [NEW]
  JULIA_ENGINE_URL=http://localhost:5000
  BRIDGE_PORT=4000

bridge/src/  [DELETED]
  Removed Phase 0 scaffold stubs (index.js, julia-bridge.js, routes/).
  Replaced by bridge/server.js.

frontend/index.html  [NEW]
  HTML entry point for Vite.

frontend/vite.config.js  [NEW]
  Vite config with React plugin, port 3000, proxy /api and /health
  to http://localhost:4000.

frontend/src/main.jsx  [NEW]
  React entry point rendering App into #root.

frontend/src/App.jsx  [IMPLEMENTED]
  Single-page scoring form with:
    - Retailer name (text, required)
    - List price (number, required)
    - Local tax rate (number, default 0)
    - Product category (text, required)
    - Card tier (select: none/one_x/one_five_x/two_x)
    - Score button with loading state
  Results panel shows:
    - Total miles, portal miles, card miles, bonus miles
    - Risk class badge (green=confirmed, yellow=uncertain, red=excluded)
    - Path, MPD
    - Full spend vector (p_list, p_portal, p_card, p_cash, v_residual)
  Error panel for bridge/engine errors.

engine/src/server.jl  [MODIFIED]
  Two fixture data changes (no engine logic changes):
    1. Added "none" CardTier with base_rate=0.0 and empty category_rates.
       When card_tier="none", card_miles = 0 as expected for users with
       no Chase United card.
    2. Added "appliances" -> uncertain to BestBuy's risk_overrides.
       Required for Phase 2 acceptance test 3 (uncertain category).
    3. Added guard: when card base_rate is 0.0 (none), skip fixture
       card rate override to preserve zero card miles.

scripts/start-dev.sh  [NEW]
  Starts Julia engine, Node bridge, Vite frontend in correct order.
  Writes PIDs to logs/pids.txt. Waits for Julia health check.

scripts/stop-dev.sh  [NEW]
  Reads PIDs from logs/pids.txt and kills them. Falls back to
  port-based kill if PID file is missing.

IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /opt/miniconda3/bin/julia (installed via conda, not juliaup).
   julialang.org is a blocked host. conda TOS must be accepted first in each
   fresh environment.

2. Always run Julia with: JULIA_PKG_SERVER=""

3. If HTTP.jl fails with "could not load symbol mbedtls_strerror":
   Run: conda install -c conda-forge "mbedtls=2.*"

4. Bridge server.js is at bridge/server.js (not bridge/src/index.js).
   The Phase 0 scaffold stubs in bridge/src/ were removed and replaced.

5. Frontend uses Vite on port 3000 with proxy to bridge on port 4000.
   No Create React App. No external UI libraries.

6. Card tier mapping (frontend -> Julia engine):
     none      -> "none"     (base_rate=0.0, card_miles=0)
     one_x     -> "explorer" (base_rate=1.0)
     one_five_x -> "club"    (base_rate=1.0, but 1.5x is semantic — see spec)
     two_x     -> "quest"    (base_rate=1.0)

7. Engine fixture changes made in this phase:
     - Added "none" card tier with base_rate=0.0
     - Added "appliances" -> uncertain risk override to BestBuy
     - Added guard to skip fixture card rate override when card base_rate=0.0
   These are fixture data additions, not engine logic changes.
   All 76 Phase 1 tests still pass.

8. The #1 invariant: card_miles = p_card * c_r(k) with NO delta factor.
   Verified in Test 3: uncertain risk class, card_miles=432 (unaffected).
