db/init.sh  [NEW]
  Idempotent database initialization script. Creates db/mileageplus.db,
  applies schema.sql, then applies seed.sql. Safe to run multiple times.

engine/src/database.jl  [NEW]
  SQLite database layer. Functions:
    - get_db_path() — returns path to db/mileageplus.db relative to repo root
    - load_latest_snapshot(db) — most recent complete snapshot or nothing
    - load_retailer(db, snapshot_id, name) — loads retailer with rates, bonuses,
      T&C rules for a given snapshot
    - load_risk_class(db, snapshot_id, retailer_name, category) — classifies
      category against T&C inclusions/exclusions
    - classify_category(category, inclusions, exclusions) — string matching for
      risk classification
    - load_process_constraints(db, retailer_name) — process constraint warnings
    - snapshot_age_hours(snapshot) — hours since snapshot completed
  Types: SnapshotInfo, RetailerData, ProcessConstraint

IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /opt/miniconda3/bin/julia (Julia 1.12.1, installed via conda)
   Always run with: JULIA_PKG_SERVER=""

2. Database initialization: bash db/init.sh
   Must be run before starting the Julia engine. The engine will exit with
   an error if db/mileageplus.db does not exist.

3. The server.jl no longer uses hardcoded fixture retailers. All retailer data
   comes from the database. The Phase 1/2 fixture retailers (Macy's, Udemy,
   Sephora) are NOT in the Phase 3 seed data — they were replaced by BestBuy,
   Nike, and Walmart to match the Phase 3 task specification.

4. Risk classification uses exact string matching against comma-separated
   inclusions/exclusions in tc_rules. Case-insensitive. The AI-based T&C
   parser is Phase 5.

5. The bridge (bridge/server.js) was NOT modified. It already passes through
   the full response body from Julia, so new fields (snapshot_id,
   snapshot_completed_at, process_constraints) are forwarded automatically.

6. The bridge converts frontend tax_rate from percentage to decimal
   (divides by 100). So tax_rate=7 in the frontend request becomes 0.07
   in the Julia request. When calling the Julia engine directly (curl),
   pass tax_rate as a decimal (e.g., 0.08).

7. Card tier aliases: the server now handles both internal names (explorer,
   quest, club) and frontend names (one_x, two_x, one_five_x).

8. The db/mileageplus.db file is NOT checked into git (.gitignore should
   exclude it). It must be regenerated via db/init.sh on each fresh clone.
