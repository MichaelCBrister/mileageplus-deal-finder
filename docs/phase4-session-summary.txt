IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /opt/miniconda3/bin/julia (Julia 1.12.1, installed via conda)
   Always run with: JULIA_PKG_SERVER=""

2. Database initialization: bash db/init.sh
   Must be run before starting the Julia engine.

3. The /score endpoint now supports a "path" parameter: "direct" (default),
   "mpx", or "stacked". Existing callers that don't send path get direct.

4. classify_category_from_tc in scoring.jl differs slightly from
   classify_category in database.jl: when a category is not in inclusions
   but also not in exclusions, scoring.jl returns :uncertain (portal miles
   included with warning) while database.jl returns :excluded. This was
   intentional for Phase 4: the task requires Electronics at Walmart to
   have leg2_portal > 0 in stacked path. The /score direct path still uses
   classify_category from database.jl (unchanged Phase 3 behavior). Phase 5
   should reconcile these two functions when the AI T&C parser is added.

5. ScoreResult now has mpx_miles, retailer_name, gc_source, and destination
   fields. All existing code passes default values ("", 0.0) for these.

6. The stacked path risk logic: when Leg 2 has excluded risk (category not
   eligible at destination), the stacked path degrades to MPX-only and the
   combined risk is :confirmed (no risky miles in the total). This is per
   the task specification.

7. rank_all generates up to N_direct + N_mpx + N_mpx * N_gc_eligible results.
   With 3 seed retailers (1 gc_eligible), that's 3 + 3 + 3 = 9 results.
   At 400 retailers this could be large. Phase 8 pagination may be needed.

8. The bridge converts tax_rate from percentage to decimal for BOTH /api/score
   and /api/rank. When calling Julia directly via curl, pass decimal (e.g., 0.08).

9. The frontend max-width was increased from 600px to 900px to accommodate
   the wider ranking table.
