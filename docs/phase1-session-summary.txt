ENVIRONMENT SETUP
-----------------
Julia was not installed. julialang.org and julialang-s3.julialang.org
are blocked by the network proxy.

Workaround:
  1. Downloaded Miniconda3 from repo.anaconda.com (allowed host)
  2. Installed Miniconda to /opt/miniconda3
  3. Installed Julia 1.10.4 via: conda install -c conda-forge julia=1.10
  4. Julia binary: /opt/miniconda3/bin/julia

Dependency issue:
  HTTP.jl depends on MbedTLS, which loaded the conda-installed
  libmbedcrypto.so 4.0.0. That version removed mbedtls_strerror,
  causing a load error.

  Fix: conda install -c conda-forge "mbedtls=2.*"
  Result: mbedtls downgraded to 2.27.0, HTTP.jl loads correctly.

Julia dependencies installed via:
  JULIA_PKG_SERVER="" julia --project=. -e 'using Pkg; Pkg.instantiate()'
  (JULIA_PKG_SERVER="" required to avoid 403 from pkg.julialang.org)

Registry was cloned from https://github.com/JuliaRegistries/General.git
(GitHub API is an allowed host).


FILES CREATED / MODIFIED
--------------------------

engine/src/MileagePlusDealFinder.jl  [NEW]
  Module entry point required by Julia's Pkg system.
  Includes types.jl, bonus.jl, scoring.jl and exports all public symbols.

engine/src/types.jl  [IMPLEMENTED]
  - SpendVector struct with 5 fields: p_list, p_portal, p_card, p_cash,
    v_residual
  - SpendVector convenience constructor: takes p_list, tax_rate (default
    0.08), optional gc_denom for gift card paths
  - RiskClass enum: confirmed, uncertain, excluded
  - PathType enum: direct, mpx, stacked
  - BonusSemantic enum: total, incremental, up_to, flat_bonus
  - CardTier struct + card_rate() lookup function
  - Retailer struct
  - Abstract BonusOffer with 3 concrete subtypes:
      FlatTieredBonus     — highest-qualifying-tier flat miles
      RateMultiplierBonus — modifies portal earn rate via semantics dispatch
      PerOrderFlatBonus   — flat miles per order if min_spend met
  - ScoreResult struct

engine/src/bonus.jl  [IMPLEMENTED]
  - compute_bonus(::FlatTieredBonus, spend, base_rate)
  - compute_bonus(::RateMultiplierBonus, spend, base_rate)
      dispatches on semantics:
        :total       -> max(0, rate - base_rate) * spend
        :incremental -> rate * spend
        :up_to       -> max(0, rate - base_rate) * spend
        :flat_bonus  -> rate (flat miles)
  - compute_bonus(::PerOrderFlatBonus, spend, base_rate)
  - total_bonus_miles(bonuses, spend, base_rate) — sums over a vector

engine/src/scoring.jl  [IMPLEMENTED]
  - risk_class(retailer, product_query) -> RiskClass
  - portal_delta(rc) -> Float64
      confirmed  -> 1.0
      uncertain  -> 1.0 (included with warning; risk_class column warns user)
      excluded   -> 0.0
  - score_direct(retailer, spend, card, bonuses; product_query) -> ScoreResult
      CRITICAL INVARIANT: card_miles = p_card * c_r(k), NO delta factor
      portal_miles = delta * p_portal * base_rate
      bonus_miles  = 0 when excluded, total_bonus_miles otherwise
      MPX and Stacked paths NOT implemented (Phase 4)

engine/src/server.jl  [IMPLEMENTED]

One bug found and fixed during testing:
  bonus.jl had a second overload for total_bonus_miles(::Vector{BonusOffer}, ...)
  that always returned 0.0. Julia dispatched this more-specific method over the
  generic Vector{<:BonusOffer} version when called with heterogeneous bonus lists.
  Fixed by removing the extraneous overload.

IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /opt/miniconda3/bin/julia (installed via conda, not juliaup)
   julialang.org is a blocked host in this environment.

2. Always run Julia with: JULIA_PKG_SERVER=""
   pkg.julialang.org is also blocked; without this env var, Pkg hangs on
   first use trying to contact the package server.

3. If HTTP.jl fails with "could not load symbol mbedtls_strerror":
   Run: conda install -c conda-forge "mbedtls=2.*"
   This downgrades mbedtls from 4.x (conda default) to 2.x (Julia-compatible).

4. If Julia tests hang during precompilation, clean stale pidfiles:
   find ~/.julia/compiled -name "*.pidfile" -delete

5. The #1 invariant in the codebase: card_miles = p_card * c_r(k) with NO
   delta factor. Portal risk class NEVER affects card mile earning.
   Tested explicitly in test_scoring.jl.
