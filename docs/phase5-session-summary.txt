bridge/tc-parser.js  [NEW]
  Claude API-powered T&C and bonus offer parser module. Exports two async
  functions:
    - parseTAndC(rawText, retailerName) — parses T&C text into structured
      inclusions/exclusions with confidence and notes
    - parseBonus(rawText, retailerName, baseRate) — parses bonus offer text
      into typed BonusOffer config (flat_tiered, rate_multiplier, per_order_flat)
  Uses claude-sonnet-4-20250514 model. Includes JSON retry logic and fallback
  on parse failure. Never throws unhandled exceptions.

  Added two new endpoints:
    - POST /api/parse-tc — parses T&C text, updates tc_rules in SQLite

  Added dependencies: path, better-sqlite3, tc-parser.js
  Both endpoints return 400 with api_key_missing error if ANTHROPIC_API_KEY
  is not set. Both update the database and return db_updated boolean.

IMPORTANT NOTES FOR FUTURE SESSIONS
--------------------------------------
1. Julia binary: /usr/local/bin/julia (Julia 1.12.5)
   Always run with: JULIA_PKG_SERVER=""

2. Database initialization: rm -f db/mileageplus.db && bash db/init.sh
   The init.sh script uses INSERT OR IGNORE, so re-running on an existing
   DB with different seed data may not update existing rows. Delete the DB
   first for a clean start.

3. RISK CLASSIFICATION RECONCILIATION: classify_category_from_tc in
   scoring.jl now delegates to classify_category in database.jl. The single
   authoritative rule is:
     - Category not in non-empty inclusions list → :excluded (NOT :uncertain)
     - :uncertain is ONLY for: confidence < 0.8, empty category, both
       inclusions and exclusions empty (no T&C data)
   This means any category not explicitly listed in a retailer's inclusions
   will have portal miles zeroed. Phase 6 scraper/parser results must
   explicitly include all eligible categories in inclusions.

4. ELECTRONICS AT WALMART: After reconciliation, "Electronics" is :excluded
   at Walmart because Walmart's inclusions are "General Merchandise,Grocery".
   For stacked path tests involving Electronics at Walmart to produce non-zero
   Leg 2 portal miles, "Electronics" must be explicitly added to Walmart's
   tc_rules inclusions. This is correct per the spec — the seed data
   accurately reflects that Walmart's portal earnings cover general
   merchandise and grocery, not electronics specifically.

5. ANTHROPIC_API_KEY: Required for /api/parse-tc and /api/parse-bonus
   endpoints. Both endpoints return 400 with api_key_missing error if the
   key is not set. The test-tc-parsing.js script also requires this key.
   Set it in the shell before starting the bridge server.

6. better-sqlite3: Added to bridge/package.json. Must run npm install in
   bridge/ directory after fresh clone. The parse endpoints use synchronous
   SQLite access to update tc_rules and bonus_offers after parsing.

7. The bridge opens a new SQLite connection for each parse request. This is
   acceptable for the low-volume on-demand parsing use case. If Phase 6
   batch parsing needs higher throughput, consider connection pooling.

8. The tc-parser.js module uses claude-sonnet-4-20250514 model. Both
   parseTAndC and parseBonus include JSON retry logic (one retry with
   clarifying instruction) and fallback objects with confidence=0.0 on
   complete failure. They never throw unhandled exceptions.
